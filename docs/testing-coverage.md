# 游戏规则覆盖率口径

## 代码覆盖门禁

`npm run test:coverage` 会把完整规则测试集打包到同一进程，运行确定性单元测试、武将与装备组合矩阵、牌局混沌测试和合法动作分叉测试，并要求 `lib/game-v2.ts` 与 `lib/game-v2-data.ts` 的语句、分支、函数和行覆盖率全部达到 100%。任何指标低于 100% 都会返回非零退出码。

测试不得使用 `c8 ignore`、降低分母或排除规则源文件来通过门禁。无法经公开动作自然到达的损坏状态保护分支，由 `gameV2TestHooks` 直接调用并真实执行。

## 行为覆盖门禁

行为覆盖率采用有限、可审计的规则契约分母，而不是宣称枚举无限长的所有牌序。当前契约包括：

- 25 名武将定义及按技能类型要求的全部 edge-case 轴；
- 108 张实体牌、全部牌名与 17 种装备；
- 34 种 pending 决策的到达、超时安全动作、越权/伪造/过期提交拒绝；
- 20 种规则帧、6 个阶段、2 人及 4–10 人共 8 种支持人数；
- 25×25 有序武将组合在【杀】与【决斗】下的接受/拒绝结算；
- 25 名武将与每种装备（含无装备基线）的边界动作。

`character-system-audit.test.ts` 会从实际规则注册表计算分母并输出 `GAME_V2_BEHAVIOR_COVERAGE=100.00`。相关组合测试、混沌测试或注册表完整性测试任一失败，完整覆盖命令即失败。

这里的 100% 表示上述已声明契约与代码控制流全部有可执行测试，不等于数学上证明所有无限牌序绝无缺陷。发布表述仍应是“无已知 Bug，全部已声明高风险状态、回归门禁和压力测试通过”。
