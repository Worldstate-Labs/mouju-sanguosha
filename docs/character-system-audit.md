## 本轮发现并修复

1. 【闭月】拒绝后会在结束阶段重复询问；现以阶段标记保证只询问一次。
2. 【观星】在牌堆和弃牌堆均为空时会创建无合法选项的 pending；现记录安全空操作并继续阶段。
3. 无牌可判定时，【洛神】【刚烈】【乐不思蜀】【闪电】【铁骑】【护驾八卦】【响应八卦】【八卦阵】八条续接路径可能停栈；现全部有确定性失败续接。
4. 【洛神】停止后可能重复进入准备/摸牌阶段；现只恢复一次正常阶段序列。
5. 【丈八蛇矛】曾绕过每回合一次【杀】限制，且出牌、响应、激将、借刀杀人日志缺少两张实体牌来源；现限制、咆哮例外和来源均有回归。
6. 【武圣】【龙胆】【奇袭】【国色】及方天画戟多目标转换曾丢失技能/原牌来源；支付武器或进攻马作为转换成本后，距离和攻击范围也未按支付后状态重算。
7. 【闪电】在伤害结算前过早离开处理区，导致【奸雄】无法获得造成伤害的实体闪电；现延迟到伤害触发完成后清理。
8. 突袭、裸衣、英姿、伤害触发、流离和多种装备技能的拒绝分支曾存在重复询问或错误续接；现逐条执行接受/拒绝双分支。
9. 模板动作只按 `skill` 匹配，曾允许动作类型混淆；精确动作又错误依赖 JSON 字段顺序。现校验动作类型、重复牌/目标、候选范围，并规范化字段顺序且仅忽略传输元数据。
10. 【无双】的两次【闪】之间没有重置八卦尝试状态；黑色八卦失败后用实体【闪】会阻止第二次八卦。现每个顺序响应段独立重置。
11. 【流离】曾晚于【铁骑】询问，且不能转给方天画戟已有目标；转移后同一目标的两个独立目标次数也可能互相污染。现按成为目标/指定目标时机排序，并逐目标次数结算。
12. 【流离】把【杀】转给方天画戟已有目标、该目标先阵亡时，原目标次数会再次向阵亡角色发决策；现安全跳过并继续后续目标。
13. 当前回合角色在【刚烈】等嵌套结算中阵亡时，身份局曾清空整个栈，使在途实体牌永久滞留；现只移除阶段帧，完成牌结算后再移交回合。
14. 多目标牌的来源阵亡后仍会触发其【铁骑】【无双】或装备技能；现阵亡武将技能失效，但在途牌的剩余目标继续结算。【无双】覆盖八卦拒绝、红色成功、无牌判定和普通响应四条路径。
15. 阵亡来源位于末号座位时，无懈/救援行动顺序曾错误回到排序首项而非来源后的首个存活座位；现同时覆盖中间座位与末位绕回 0 号座位。
16. 嵌套伤害直接触发胜利时，处理区、判定展示区或触发队列可能残留；终局现在原子清理所有临时状态。
17. 多名【鬼才】、最终【天妒】、无懈中的【集智】、无双两闪之间的【连营】和一次失去两件装备的两次【枭姬】均加入顺序与实体牌回归。

每个修复均有确定性回归测试，关键条件另有变异测试；状态机修改还经过独立新 seed 混沌批次。当前结论是：在上述规则集、人数、状态和压力范围内无已知 Bug，最终收敛轮未发现新缺口；这不等同于数学意义上的绝对零 Bug。

## 已知边界

- 未命中的少量源码分支均是合法动作白名单之前或之后的私有防御异常，例如篡改双人选色、重复选择已入阵武将、伪造寒冰剑 zone、注入未知规则帧；公开 API 的拒绝路径已有测试，但这些私有 handler 异常本身不可由合法状态到达。
- 覆盖率、变异测试和有限 seed 探索不能穷举无限长的所有牌序与所有并发时序；新增规则或状态机修改仍必须增加定向回归并使用全新 seed 区间。
- 本轮发布前已完成完整构建、artifact validation 和浏览器核心旅程；后续若继续改动，仍须重跑这些门禁并以对应源码提交创建新的 Sites 生产检查点。
